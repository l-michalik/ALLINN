# AGENTS.md

## Working Style

- Implement simple, readable, minimal solutions that fit the existing repository structure.
- Do not add defensive logic without a concrete reason.
- Add explicit typing where it improves readability, safety, or module APIs.
- Prefer straightforward code over clever helpers, abstractions, and hidden magic.
- Preserve existing naming, formatting, and file organization.

## Before Editing

- Inspect the relevant files first.
- State assumptions when behavior is ambiguous.
- Avoid changing unrelated files.
- Do not rewrite history or revert user changes unless explicitly asked.

## Validation

- For frontend changes, verify the UI in a browser when a local target is known.
- Dont write tests

## Git

- Keep commits focused.
- Do not amend commits unless explicitly requested.
- Do not use destructive git commands without explicit approval.
